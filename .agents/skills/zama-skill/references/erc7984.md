## Overview

ERC-7984 is the standard for confidential tokens on FHEVM. It defines a fungible token interface
where balances and transfer amounts are encrypted using FHE, while maintaining compatibility with
the broader ERC-20 ecosystem through wrapper contracts.

The reference implementation is in `@openzeppelin/confidential-contracts`.

---

## Key Differences from ERC-20

| Feature | ERC-20 | ERC-7984 |
|---------|--------|----------|
| Balance type | `uint256` | `euint64` |
| Transfer amount | `uint256` | `externalEuint64 + inputProof` (or `euint64` internally) |
| Max decimals | 18 | **6** (to fit euint64 range) |
| Approval system | `approve(spender, amount)` | Operator-based (time-bounded via `uint48` timestamp) |
| Balance visibility | Public | Encrypted (only owner can decrypt) |
| Transfer function | `transfer` / `transferFrom` | `confidentialTransfer` / `confidentialTransferFrom` |
| Transfer return | `bool` | `euint64` (the transferred amount) |

## Using OpenZeppelin Confidential Contracts

### Installation

```bash
# Pin to ^0.4.0. @openzeppelin/confidential-contracts@0.4.0 peers @fhevm/solidity = "0.11.1"
# exact; an unpinned install will jump if a future major re-pegs to a newer @fhevm/solidity.
npm install @openzeppelin/confidential-contracts@^0.4.0
```

### Basic ERC-7984 Token

```solidity
/* Project default is ^0.8.28; ^0.8.27 is the minimum when importing
   @openzeppelin/confidential-contracts (ERC7984.sol declares ^0.8.27).
   Stay on ^0.8.28 unless a dependency forces lower. */
pragma solidity ^0.8.28;

import { FHE, euint64, externalEuint64 } from "@fhevm/solidity/lib/FHE.sol";
import { ZamaEthereumConfig } from "@fhevm/solidity/config/ZamaConfig.sol";
import { ERC7984 } from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";

contract ConfidentialToken is ZamaEthereumConfig, ERC7984 {
    /* Constructor takes 3 params: name, symbol, contractURI */
    constructor() ERC7984("Confidential Token", "CTKN", "https://example.com/metadata.json") {}

    function mint(externalEuint64 encAmount, bytes calldata inputProof) external {
        /* CRITICAL: _mint takes a verified internal euint64 handle (already proven and
           registered with the ACL via FHE.fromExternal), NOT externalEuint64 + inputProof.
           Call FHE.fromExternal() yourself first.
           The base contract's _mint signature (ERC7984.sol:242):
             function _mint(address to, euint64 amount) internal returns (euint64 transferred)
           The returned `transferred` handle is the ACL-allowed amount that actually moved. */
        euint64 amount = FHE.fromExternal(encAmount, inputProof);
        FHE.allowThis(amount);
        _mint(msg.sender, amount);
    }
}
```

### Available Extensions

| Extension | Purpose |
|-----------|---------|
| `ERC7984ERC20Wrapper` | Wrap existing ERC-20 tokens as confidential |
| `ERC7984Freezable` | Freeze/unfreeze accounts (encrypted frozen amounts) |
| `ERC7984ObserverAccess` | Grant read-only access to specific observers |
| `ERC7984Omnibus` | Omnibus account pattern for custodians |
| `ERC7984Restricted` | KYC/whitelist restrictions (3-state: DEFAULT/BLOCKED/ALLOWED) |
| `ERC7984Rwa` | Real World Asset tokenization (roles, freeze, restrict, pause) |
| `ERC7984Votes` | Governance voting with encrypted balances |

### ERC-20 Wrapper Example

```solidity
pragma solidity ^0.8.28;

import { ZamaEthereumConfig } from "@fhevm/solidity/config/ZamaConfig.sol";
import { ERC7984ERC20Wrapper } from "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984ERC20Wrapper.sol";
import { IERC20 } from "@openzeppelin/contracts/token/ERC20/IERC20.sol";

contract WrappedConfidential is ZamaEthereumConfig, ERC7984ERC20Wrapper {
    /* ERC7984ERC20Wrapper constructor takes 1 param: the underlying ERC20.
       ERC7984 constructor takes 3 params: name, symbol, contractURI. */
    constructor(IERC20 underlying)
        ERC7984ERC20Wrapper(underlying)
        ERC7984("Wrapped Confidential", "wCTKN", "https://example.com/wrapped.json")
    {}
}
```

## Operator System (Time-Bounded, NOT Boolean)

ERC-7984 uses a **time-bounded operator model** instead of ERC-20 allowances.
Operators are authorized until a specific timestamp (`uint48`), not with a boolean on/off.

```solidity
/* Grant operator permission until a specific timestamp */
function setOperator(address operator, uint48 until) public virtual;

/* Check if operator is currently authorized for holder.
   Returns true if holder == spender OR block.timestamp <= until */
function isOperator(address holder, address spender) public view virtual returns (bool);

/* Transfer as authorized operator */
function confidentialTransferFrom(
    address from,
    address to,
    externalEuint64 encAmount,
    bytes calldata inputProof
) external returns (euint64 transferred);

/* Also available with already-encrypted amount: */
function confidentialTransferFrom(
    address from,
    address to,
    euint64 amount
) external returns (euint64 transferred);
```

**Usage example:**
```solidity
/* Grant operator access for 30 days */
uint48 expiry = uint48(block.timestamp + 30 days);
token.setOperator(operatorAddress, expiry);

/* Revoke operator access (set expiry to 0 or past timestamp) */
token.setOperator(operatorAddress, 0);

/* Check if still authorized */
bool authorized = token.isOperator(holderAddress, operatorAddress);
```

**Event:** `event OperatorSet(address indexed holder, address indexed operator, uint48 until);`

## Transfer Functions (8 Variants)

All eight transfer variants are declared `public virtual` in the implementation
(`ERC7984.sol:115-199`) and return `euint64 transferred` (NOT `bool`). The `IERC7984` interface
declares them `external`; the base implementation widens to `public`.

| Function | Params | Purpose |
|----------|--------|---------|
| `confidentialTransfer(to, encAmount, proof)` | external input | Direct transfer (verifies proof) |
| `confidentialTransfer(to, amount)` | euint64 | Direct transfer (caller already has ACL on `amount`) |
| `confidentialTransferFrom(from, to, encAmount, proof)` | external input | Operator transfer (verifies proof) |
| `confidentialTransferFrom(from, to, amount)` | euint64 | Operator transfer (caller already has ACL on `amount`) |
| `confidentialTransferAndCall(to, encAmount, proof, data)` | external + callback | Transfer + hook |
| `confidentialTransferAndCall(to, amount, data)` | euint64 + callback | Transfer + hook |
| `confidentialTransferFromAndCall(from, to, encAmount, proof, data)` | external + callback | Operator + hook |
| `confidentialTransferFromAndCall(from, to, amount, data)` | euint64 + callback | Operator + hook |

## Disclose Pattern (2-Step)

ERC-7984 has a built-in 2-step disclose pattern for revealing encrypted amounts:

### Step 1: Request disclosure (on-chain)
```solidity
/* Caller must have ACL access to the encrypted amount.
   Marks it as publicly decryptable and emits AmountDiscloseRequested. */
function requestDiscloseEncryptedAmount(euint64 encryptedAmount) public;
```

### Step 2: Finalize disclosure (after off-chain decrypt)
```solidity
/* Verifies the decryption proof and emits AmountDisclosed with the cleartext. */
function discloseEncryptedAmount(
    euint64 encryptedAmount,
    uint64 cleartextAmount,
    bytes calldata decryptionProof
) public;
```

**Events** (verified against `@openzeppelin/confidential-contracts@0.4.0` `IERC7984.sol`):
- `event AmountDiscloseRequested(euint64 indexed encryptedAmount, address indexed requester);`
- `event AmountDisclosed(euint64 indexed encryptedAmount, uint64 amount);` /* second param NOT indexed; field name is `amount` */

Between steps 1 and 2, the dApp calls `instance.publicDecrypt([handle])` off-chain using the
relayer SDK, then submits the cleartext + proof back to `discloseEncryptedAmount()`.

## ERC7984Rwa (Real World Asset) Extension

The RWA extension inherits `ERC7984Freezable + ERC7984Restricted + Pausable + Multicall + AccessControl`
(`ERC7984Rwa.sol:22`):

```solidity
import { ERC7984Rwa } from "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984Rwa.sol";

contract MyRWAToken is ZamaEthereumConfig, ERC7984Rwa {
    constructor()
        ERC7984("RWA Token", "RWA", "https://example.com/rwa.json")
        ERC7984Rwa(msg.sender)  /* admin address */
    {}
}
```

**Admin/role-introspection (`ERC7984Rwa.sol:59-77`):**
| Function | Purpose |
|----------|---------|
| `addAgent(account)` | Grant AGENT_ROLE - `onlyAdmin` (`ERC7984Rwa.sol:69-72`) |
| `removeAgent(account)` | Revoke AGENT_ROLE - `onlyAdmin` (`ERC7984Rwa.sol:74-77`) |
| `isAdmin(account) -> bool` | View admin status - public, no role gate (`ERC7984Rwa.sol:59`) |
| `isAgent(account) -> bool` | View agent status - public, no role gate (`ERC7984Rwa.sol:64`) |

**Agent functions (require AGENT_ROLE, all returning `euint64` where applicable):**
| Function | Purpose |
|----------|---------|
| `confidentialMint(to, encAmount, proof) -> euint64` | Mint new tokens (verifies proof) |
| `confidentialMint(to, euint64 encryptedAmount) -> euint64` | Mint with pre-verified handle |
| `confidentialBurn(account, encAmount, proof) -> euint64` | Burn tokens (verifies proof) |
| `confidentialBurn(account, euint64 encryptedAmount) -> euint64` | Burn with pre-verified handle |
| `forceConfidentialTransferFrom(from, to, encAmount, proof) -> euint64` | Force transfer (verifies proof) |
| `forceConfidentialTransferFrom(from, to, euint64 encryptedAmount) -> euint64` | Force transfer with pre-verified handle |
| `blockUser(account)` | Block user from transacting |
| `unblockUser(account)` | Reset user to DEFAULT (NOT strict ALLOWED) |
| `setConfidentialFrozen(account, encAmount, proof)` | Freeze an amount on account (verifies proof) |
| `setConfidentialFrozen(account, euint64 encryptedAmount)` | Freeze with pre-verified handle |
| `pause()` / `unpause()` | Emergency pause/unpause |

**Force transfer scope (`ERC7984Rwa.sol:160-185, 223`):** `forceConfidentialTransferFrom` bypasses
the pause check and the `from`-side restriction check ONLY. The `to`-side restriction check is
still enforced; frozen tokens are NOT bypassed and must be unfrozen first.

**Restriction system (3-state):**
- `DEFAULT` - no explicit restriction set
- `BLOCKED` - user explicitly blocked (cannot transact)
- `ALLOWED` - user explicitly allowed

## Important Constraints

1. **Max 6 decimals**: `euint64` max value is ~1.8e19. With 18 decimals, max supply would be ~18 tokens.
   Use 6 decimals for practical token supplies.

2. **No public totalSupply**: Use `confidentialTotalSupply()` which returns `euint64`.
   To reveal it, use the disclose pattern above.

3. **Overflow wraps silently**: FHE arithmetic is unchecked. Design mint/transfer logic to prevent overflow
   using `FHE.select` with `FHE.le` guards.

4. **ACL management**: The ERC7984 base contract handles ACL for balances internally. Do not add manual `FHE.allow` calls on balance handles inside standard transfer operations. Only add `FHE.allow` for values you add on top of the base contract.

5. **`_mint` takes `euint64`, NOT external types**: The internal `_mint(address, euint64)` function
   expects an already-decrypted encrypted value. Call `FHE.fromExternal()` first in your public mint function.

## Critical Gotchas

> Non-obvious ERC-7984 behaviors that agents consistently get wrong. Applies to `@openzeppelin/confidential-contracts@^0.4.0`.

### 1. Constructor Takes 3 Parameters, NOT 2

```solidity
/* WRONG - missing contractURI */
constructor() ERC7984("Token", "TKN") {}

/* CORRECT - 3 params: name, symbol, contractURI */
constructor() ERC7984("Token", "TKN", "https://example.com/metadata.json") {}
```

### 2. `_mint` Takes `euint64`, NOT `externalEuint64 + inputProof`

```solidity
/* WRONG - _mint does NOT accept external types */
_mint(msg.sender, encAmount, inputProof);

/* CORRECT - convert first, then mint */
euint64 amount = FHE.fromExternal(encAmount, inputProof);
FHE.allowThis(amount);
_mint(msg.sender, amount);
```

### 3. `setOperator` Takes `uint48` Timestamp, NOT `bool`

```solidity
/* WRONG - no bool parameter */
token.setOperator(operator, true);

/* CORRECT - uint48 expiry timestamp */
token.setOperator(operator, uint48(block.timestamp + 30 days));
```

### 4. Transfer Functions Are `confidentialTransfer`/`confidentialTransferFrom`, NOT `transfer`/`transferFrom`

All return `euint64 transferred`, NOT `bool`.

### 5. Extension Name Is `ERC7984Rwa`, NOT `ERC7984RWA`

The "wa" in "Rwa" is lowercase. Using `ERC7984RWA` will fail to import.

### 6. `ERC7984ERC20Wrapper` Constructor Takes Only the Underlying Token

```solidity
/* WRONG - wrapper constructor does NOT take name/symbol */
ERC7984ERC20Wrapper(underlying, "Name", "SYM")

/* CORRECT - wrapper takes 1 param, ERC7984 takes 3 params separately */
ERC7984ERC20Wrapper(underlying) ERC7984("Name", "SYM", "uri")
```

### 7. `isOperator` Params Are `(holder, spender)`, NOT `(owner, operator)`

The parameter names matter for clarity: `isOperator(holder, spender)` checks if `spender`
is authorized to act on behalf of `holder`.
